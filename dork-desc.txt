Tailored for bug bounty hunters, with emphasis on **OAuth misconfigurations**, **cloud (AWS Cognito)**, **APIs**, **exposed secrets**, **AI-related exposures**, and general recon — inspired by your `source.txt` materials (BruteCat AI hacking, OAuth writeups, Cognito risks, etc.).

### Core Operators Recap
- `site:target.com` — Focus on domain
- `inurl:` — Keyword in URL
- `intitle:` — Keyword in title
- `filetype:` or `ext:` — File extensions
- `intext:` / `allintext:` — Content search
- Combine with `-github.com -stackoverflow.com` to reduce noise

**Git Dorking + Shodan Dorking Cheat Sheet (2026 Edition)**

Continuing from Google Dorking, here are powerful techniques for **Git** (exposed repos/directories + GitHub searches) and **Shodan** (internet-wide device/service scanning). These are high-value in bug bounty and recon, especially when chained with OAuth, cloud (Cognito), API, and secret exposures from your source materials.

### 1. Git Dorking / Exposed .git Directories

**Exposed `.git/` folders** are a critical misconfiguration. They allow full source code reconstruction, including secrets, configs, and history.

#### Discovery Dorks (Google + Others)
```
site:target.com inurl:/.git/ OR inurl:/.git/config
intitle:"index of" ".git"
site:target.com filetype:git OR inurl:.git/HEAD
site:target.com inurl:.git/config "core.repositoryformatversion"
inurl:/.git/ site:target.com -github.com
```

**Automated Tools**:
- `git-dumper.sh https://target.com/.git/ output/`
- `git-dumper` / `GitTools` suite
- `ffuf -u https://target.com/FUZZ -w wordlist.txt` with `.git/`, `.env`, etc.
- DotGit Chrome extension for alerts.

**Exploitation Steps** (when `.git` is exposed):
1. Download the directory recursively.
2. Run `git status`, `git log`, `git checkout`, `git fsck` to recover files.
3. Grep for secrets: `grep -r "password\|key\|secret\|AKIA" .`

**Impact**: Source code leak → hardcoded OAuth secrets, DB creds, AWS keys → Account Takeover or RCE.

### 2. GitHub Dorking (for Secrets & OAuth Leaks)

GitHub advanced search operators are excellent for finding leaked keys across public repos.

#### Core GitHub Search Operators
- `filename:.env`
- `extension:env`
- `path:.github` (workflows often contain secrets)
- `language:yaml` or `language:json`
- `org:targetorg` (if known)

#### High-Impact GitHub Dorks for Secrets
```
filename:.env ("DB_PASSWORD" OR "AWS_SECRET" OR "CLIENT_SECRET" OR "OAUTH")
filename:.env "cognito" OR "userpool"
"AKIA" filename:.env OR filename:.js OR filename:.json (AWS keys)
"ghp_" OR "gho_" OR "github_pat_" (GitHub tokens)
"sk-" OR "pk-" OR "stripe" (payment/OAuth keys)
filename:config "client_secret" OR "redirect_uri"
inurl:settings OR inurl:secrets "oauth"
"Bearer" OR "Authorization:" extension:js OR extension:py
```

**OAuth / Cognito Specific**:
- `cognito` OR `userPoolId` OR `IdentityPoolId` extension:js OR extension:json
- `"aws_cognito"` OR `"CognitoUserPool"`

**Pro Tips**:
- Use GitHub’s advanced search + filters (stars, language, etc.).
- Tools: `gitleaks`, `trufflehog`, `git-secrets`.
- Always check forks and old commits.

### 3. Shodan Dorking (Internet-Wide Recon)

Shodan indexes banners, services, devices, and vulnerabilities — perfect for finding exposed OAuth endpoints, admin panels, cloud services, etc.

#### Core Shodan Filters
- `org:"Target Company"` — Organization filter
- `hostname:target.com` or `domain:target.com`
- `net:IP/range` — IP blocks
- `port:443` or `port:80`
- `product:Apache` / `product:Nginx`
- `country:US` / `city:NewYork`
- `ssl:"target.com"` — Certificate searches
- `http.title:"Login"` or `http.html:"oauth"`
- `http.status:200` / `http.status:403`

#### Bug Bounty / Security Focused Shodan Dorks
**General Recon**:
```
hostname:target.com
org:"Target Inc" port:443
domain:target.com http.title:"Admin" OR "Dashboard"
```

**OAuth / API / Login**:
```
http.html:"oauth" OR http.html:"authorize" OR http.html:"client_id"
http.title:"OAuth" OR http.title:"SSO" OR http.title:"Login"
port:443 "cognito" OR "userpool"
http.component:"oauth2" OR http.html:"redirect_uri"
```

**Cloud & Exposed Services**:
```
"cognito" OR "aws cognito" hostname:target.com
"amazonaws" port:443
"S3" OR "bucket" http.html
port:22 OR port:3389 (SSH/RDP — risky exposures)
```

**Vulnerable / Juicy**:
```
http.status:200 product:"phpMyAdmin"
"index of" port:80 OR port:443
"SQL syntax" OR "error" http.html
"Spring" OR "MariaDB" (for vector store / CVE hunting)
```

**Advanced Chains**:
- `org:"Target" ssl:"Target"` → Filter by certs
- Combine with `after:2025-01-01` or `before:` for recent exposures
- Use Shodan’s favicon hash for specific apps

**Tools & Workflow**:
- Shodan CLI / Web
- Combine with Nuclei for scanning results
- Export IPs → mass scanning with `httpx`, `nuclei`, etc.
- Chain with Google/Git dorks for full picture.
